cron으로 반복 작업 자동화하기

로그 정리, 백업, 헬스 체크처럼 주기적으로 반복해야 하는 작업을 cron에 맡기면 사람이 직접 실행하지 않아도 됩니다.

crontab 기본 문법

# ┌─ 분(0-59)
# │ ┌─ 시(0-23)
# │ │ ┌─ 일(1-31)
# │ │ │ ┌─ 월(1-12)
# │ │ │ │ ┌─ 요일(0-7, 0과 7 모두 일요일)
# │ │ │ │ │
  * * * * * 실행할_명령

자주 쓰는 패턴

표현식의미
0 2 * * *매일 새벽 2시
*/10 * * * *10분마다
0 9 * * 1-5평일 오전 9시
0 0 1 * *매월 1일 자정

crontab 편집과 확인

crontab -e로 현재 사용자의 crontab을 편집하고, crontab -l로 등록된 작업 목록을 확인합니다. 시스템 전체 작업은 /etc/cron.d/ 디렉토리에 파일로 추가합니다.

로그와 오류 처리

cron 작업의 출력은 기본적으로 메일로 전송되는데, 대부분의 환경에서 메일 설정이 되어 있지 않아 묵묵히 사라집니다. 로그를 남기려면 명령 끝에 >> /var/log/myjob.log 2>&1을 추가해 파일로 기록하는 것이 좋습니다.

주의사항

  • cron은 기본적으로 최소한의 환경변수만 가집니다. 스크립트 안에서 절대 경로를 사용하거나 PATH를 명시적으로 설정해야 합니다.
  • 스크립트 파일에 실행 권한(chmod +x)이 있는지 확인하세요.
