DDoS 공격의 원리와 기본 방어 방법

DDoS는 대형 서비스만의 문제가 아닙니다. 누구나 표적이 될 수 있고, 기본적인 방어는 충분히 준비할 수 있습니다.

DDoS 공격의 원리

DDoS(분산 서비스 거부)는 수많은 출처에서 동시에 대량의 트래픽을 서버에 보내 정상 사용자의 요청을 처리하지 못하게 만드는 공격입니다. 봇넷(악성코드에 감염된 수천~수만 대의 기기)을 이용하는 것이 일반적입니다.

주요 공격 유형

  • 볼류메트릭 공격: UDP/ICMP 플러드처럼 대역폭을 가득 채우는 방식
  • 프로토콜 공격: SYN 플러드처럼 서버의 연결 상태 테이블을 고갈시키는 방식
  • 애플리케이션 레이어 공격: HTTP GET/POST 플러드처럼 정상 요청처럼 보이도록 위장하는 방식

Cloudflare를 통한 방어

Cloudflare 같은 CDN/보안 서비스를 앞에 두면 오리진 서버의 실제 IP가 숨겨지고, Cloudflare가 공격 트래픽을 흡수·차단합니다. 무료 플랜에서도 기본적인 DDoS 방어가 제공됩니다.

서버 단의 기본 대응

  • rate limiting으로 단일 IP의 요청 수 제한
  • SYN 쿠키 활성화로 SYN 플러드 완화
  • 불필요한 UDP 서비스 비활성화
  • 업스트림 ISP에 블랙홀 라우팅 요청

사전 준비의 중요성

공격이 시작된 후에 대응 방법을 찾는 것은 늦습니다. 미리 CDN을 도입하고, 긴급 연락처와 대응 절차를 문서화해두는 것이 피해를 최소화하는 가장 확실한 방법입니다.
