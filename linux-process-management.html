리눅스 프로세스 관리 명령어 실전 정리

서버에서 문제가 생겼을 때 가장 먼저 확인하는 것이 프로세스 상태입니다. 몇 가지 명령어만 익혀두면 대부분의 상황을 빠르게 파악할 수 있습니다.

실행 중인 프로세스 확인

ps aux는 현재 실행 중인 모든 프로세스를 스냅샷 형태로 출력합니다. CPU와 메모리 사용량 순으로 정렬하려면 ps aux --sort=-%cpu 또는 ps aux --sort=-%mem을 사용합니다.

실시간 모니터링 — top과 htop

top은 실시간으로 프로세스 목록과 시스템 자원 사용 현황을 보여줍니다. 더 직관적인 인터페이스가 필요하다면 htop을 설치해 사용하면 색상 표시와 마우스 조작이 가능해 훨씬 편리합니다.

프로세스 종료

kill PID는 해당 PID의 프로세스에 SIGTERM 신호를 보내 정상 종료를 요청합니다. 프로세스가 응답하지 않으면 kill -9 PID로 강제 종료할 수 있으나, 데이터 손실 가능성이 있으므로 최후 수단으로 사용해야 합니다.

pkill 프로세스명은 PID 대신 프로세스 이름으로 종료 신호를 보낼 수 있어 편리합니다.

백그라운드 실행과 작업 제어

명령어 뒤에 &를 붙이면 백그라운드에서 실행됩니다. jobs로 현재 셸의 백그라운드 작업 목록을 확인하고, fg %번호로 포그라운드로 가져올 수 있습니다.

특정 포트를 점유한 프로세스 찾기

ss -tlnp | grep 8080
# 또는
lsof -i :8080

어떤 프로세스가 특정 포트를 사용하고 있는지 파악할 때 유용합니다. 포트 충돌 문제를 디버깅할 때 가장 먼저 실행해야 할 명령입니다.
