TCP와 UDP의 차이점과 선택 기준

어떤 프로토콜이 더 좋냐고 묻기 전에, 어떤 상황에서 무엇이 더 적합한지를 이해하는 것이 먼저입니다.

TCP — 신뢰성 우선

TCP(Transmission Control Protocol)는 3-way handshake로 연결을 수립하고, 모든 패킷의 순서와 도달 여부를 보장합니다. 패킷이 손실되면 재전송합니다. 웹(HTTP/HTTPS), 이메일, 파일 전송처럼 데이터 무결성이 중요한 곳에 쓰입니다.

UDP — 속도 우선

UDP(User Datagram Protocol)는 연결 수립 없이 패킷을 바로 보내고, 도달 여부나 순서를 보장하지 않습니다. 오버헤드가 적어 속도가 빠릅니다. 실시간 스트리밍, 온라인 게임, VoIP, DNS처럼 약간의 손실보다 지연이 더 문제인 경우에 사용합니다.

QUIC — TCP와 UDP의 절충

구글이 개발한 QUIC 프로토콜은 UDP 위에서 TCP의 신뢰성과 TLS 암호화를 구현합니다. HTTP/3의 기반이 되며, 패킷 손실 복구를 더 빠르게 처리해 모바일 환경에서 특히 효과적입니다.

방화벽 설정과의 관계

방화벽 규칙을 작성할 때 TCP/UDP를 구분해서 허용해야 합니다. 예를 들어 DNS는 기본적으로 UDP 53번 포트를 사용하지만, 응답이 클 경우 TCP 53번으로 전환됩니다. 두 프로토콜 모두 열어두어야 정상 동작합니다.
