Nginx/Apache 웹서버 로그 분석 기초

웹서버 로그는 서버에 무슨 일이 일어나고 있는지 알 수 있는 가장 직접적인 정보 소스입니다.

액세스 로그 형식 이해

Nginx의 기본 액세스 로그는 combined 형식으로, 각 줄에 클라이언트 IP, 요청 시간, HTTP 메서드, URL, 응답 코드, 전송 바이트 수, 리퍼러, 사용자 에이전트 순서로 기록됩니다.

203.0.113.5 - - [01/May/2026:09:12:33 +0900] "GET /index.html HTTP/1.1" 200 4523

자주 요청된 URL 파악

awk '{print $7}' /var/log/nginx/access.log \
  | sort | uniq -c | sort -rn | head -20

어떤 페이지가 가장 많이 요청되는지 빠르게 파악할 수 있습니다. 비정상적으로 특정 경로의 요청이 급증한다면 스캔 공격의 징후일 수 있습니다.

HTTP 상태 코드별 집계

awk '{print $9}' /var/log/nginx/access.log \
  | sort | uniq -c | sort -rn

500 오류가 갑자기 늘었다면 애플리케이션 장애, 404가 많다면 잘못된 링크나 크롤러 스캔, 301/302가 많다면 리다이렉트 설정을 점검해야 합니다.

에러 로그 확인

에러 로그는 /var/log/nginx/error.log에서 확인합니다. tail -f로 실시간 모니터링하면 배포 직후 오류를 즉시 포착할 수 있습니다.

로그 로테이션

로그 파일은 계속 쌓이므로 logrotate를 설정해 일정 크기나 기간이 지나면 자동으로 압축·보관·삭제하도록 관리해야 합니다. Nginx는 설치 시 기본 logrotate 설정이 포함되어 있습니다.
