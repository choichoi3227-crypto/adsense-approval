서버 운영 2026.05.24

서버 보안 강화 체크리스트 15가지

보안은 한 번 설정하고 끝이 아니지만, 초기 설정을 제대로 해두면 이후 운영이 훨씬 안전해집니다.

기본 접근 제어

  • ☑ root 직접 로그인 차단 (PermitRootLogin no)
  • ☑ SSH 비밀번호 인증 비활성화, 키 기반 인증만 허용
  • ☑ SSH 기본 포트(22) 변경
  • ☑ 불필요한 사용자 계정 삭제
  • ☑ 최소 권한 원칙 적용(sudo 범위 제한)

방화벽 설정

  • ☑ 기본 인바운드 트래픽 차단
  • ☑ 필요한 포트만 화이트리스트 허용
  • ☑ 공격 차단 도구(Fail2ban) 활성화
  • ☑ 내부 관리 포트는 특정 IP로만 제한

소프트웨어 업데이트

  • ☑ 운영체제 패키지 최신화
  • ☑ 자동 보안 업데이트 활성화 (unattended-upgrades)
  • ☑ 사용하지 않는 서비스/패키지 제거

감사 및 모니터링

  • ☑ 로그인 시도 로그 모니터링 설정
  • ☑ 중요 시스템 파일 변경 감지(auditd 또는 AIDE)
  • ☑ 비정상적인 프로세스 및 네트워크 연결 주기적 점검

데이터 보호

  • ☑ 정기 백업 자동화 및 복구 테스트
  • ☑ 민감 데이터 파일 권한 최소화
  • ☑ 환경 변수나 설정 파일에 비밀번호 평문 저장 금지
