서버 성능 튜닝 입문: 병목을 찾는 법

성능 문제는 무조건 서버 사양을 올린다고 해결되지 않습니다. 먼저 어디가 병목인지 정확히 찾아야 합니다.

CPU 병목 확인

top 또는 mpstat 1으로 CPU 사용률을 확인합니다. 특정 프로세스가 CPU를 독점하고 있다면 해당 코드 최적화나 수평 확장을 검토해야 합니다. wa(I/O 대기) 수치가 높다면 CPU보다 디스크가 문제입니다.

메모리 병목 확인

free -h로 메모리 여유 공간을 확인합니다. 스왑 사용량이 계속 높아지고 있다면 메모리 부족 상태로, 프로세스 메모리 사용을 줄이거나 메모리를 증설해야 합니다.

vmstat 1의 si, so 항목(스왑 입출력)이 지속적으로 0이 아니라면 심각한 메모리 부족 징후입니다.

디스크 I/O 병목 확인

iostat -xz 1
iotop

await 값이 크고 %util이 100에 가깝다면 디스크가 포화 상태입니다. 읽기 캐싱을 강화하거나 SSD로 교체, 또는 I/O를 여러 디스크로 분산하는 것을 고려합니다.

네트워크 병목 확인

iftop이나 nethogs로 어떤 프로세스가 네트워크 대역폭을 많이 사용하는지 확인합니다. 내부 서비스 간 트래픽이 많다면 데이터 구조나 API 호출 빈도를 재검토할 필요가 있습니다.

튜닝 전 베이스라인 측정

무엇을 바꾸기 전에 반드시 현재 상태를 수치로 기록해 두어야 합니다. 변경 전후를 비교할 수 없으면 튜닝이 효과가 있었는지 알 수 없습니다.
