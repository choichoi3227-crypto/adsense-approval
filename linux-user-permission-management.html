리눅스 사용자와 권한 관리 완전 정리

리눅스 권한 시스템은 처음에 낯설지만, 규칙만 이해하면 서버를 훨씬 안전하게 운영할 수 있습니다.

사용자와 그룹 개념

리눅스의 모든 파일과 프로세스는 소유자(user)와 그룹(group)을 가집니다. 권한은 소유자, 그룹 구성원, 그 외 사용자(other) 세 범주에 각각 읽기(r), 쓰기(w), 실행(x) 권한으로 나뉩니다.

사용자 계정 관리

# 사용자 생성
adduser deploy

# sudo 권한 부여
usermod -aG sudo deploy

# 사용자 목록 확인
cat /etc/passwd

# 사용자 삭제 (홈 디렉토리 포함)
deluser --remove-home olduser

파일 권한 변경 — chmod

권한은 숫자(8진수)나 기호로 표현합니다. chmod 755 script.sh는 소유자에게 rwx, 그룹과 others에게 r-x를 부여합니다.

숫자의미
7rwx (읽기+쓰기+실행)
6rw- (읽기+쓰기)
5r-x (읽기+실행)
4r-- (읽기 전용)

소유자 변경 — chown

# 소유자만 변경
chown deploy /var/www/app

# 소유자와 그룹 함께 변경
chown deploy:www-data /var/www/app

# 하위 디렉토리까지 재귀 적용
chown -R deploy:www-data /var/www/app

sudo 권한 세밀하게 제어하기

visudo로 /etc/sudoers를 편집하면 특정 사용자가 특정 명령만 sudo로 실행할 수 있도록 제한할 수 있습니다. 모든 권한을 주는 것보다 필요한 명령만 허용하는 것이 더 안전합니다.
