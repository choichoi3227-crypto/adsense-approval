스왑(Swap) 메모리란 무엇이고 어떻게 설정하나

스왑은 물리 메모리가 부족할 때 디스크를 임시 메모리처럼 활용하는 기술입니다. 만능 해결책은 아니지만, 적절히 구성하면 갑작스러운 메모리 부족 상황에서 서버를 지킬 수 있습니다.

스왑의 역할

운영체제는 물리 메모리가 부족해지면 잘 사용되지 않는 메모리 페이지를 디스크의 스왑 공간으로 내보냅니다. 이를 통해 즉각적인 OOM(Out of Memory) 강제 종료를 늦출 수 있지만, 디스크 I/O가 수반되므로 속도는 크게 느려집니다.

스왑 파일 생성

fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

부팅 시 자동 활성화하려면 /etc/fstab에 /swapfile none swap sw 0 0 줄을 추가해야 합니다.

swappiness 조정

vm.swappiness는 커널이 스왑을 얼마나 적극적으로 사용할지 결정하는 값으로, 0~100 사이입니다. 기본값은 60이며, 서버 환경에서는 10 정도로 낮춰 가능한 한 물리 메모리를 우선 사용하도록 설정하는 것이 일반적입니다.

echo 'vm.swappiness=10' >> /etc/sysctl.conf
sysctl -p

클라우드 환경에서의 고려사항

AWS EC2 등 클라우드 인스턴스는 기본적으로 스왑이 비활성화되어 있습니다. 메모리가 작은 인스턴스에서는 스왑을 설정해두면 예기치 않은 메모리 급증 상황에서 시간을 벌 수 있습니다. 다만 스왑이 자주 사용된다면 인스턴스 타입 업그레이드가 근본적인 해결책입니다.
