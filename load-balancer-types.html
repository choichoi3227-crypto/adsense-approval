로드 밸런서 종류와 동작 방식 정리

하나의 서버가 감당하기 어려운 트래픽을 여러 서버로 나누는 로드 밸런서, 어떻게 동작하는지 알면 장애 대응도 빨라집니다.

L4 vs L7 로드 밸런서

L4(전송 계층) 로드 밸런서는 IP 주소와 포트 번호만 보고 트래픽을 분배합니다. 처리 속도가 빠르지만 애플리케이션 내용은 알 수 없습니다.

L7(애플리케이션 계층) 로드 밸런서는 HTTP 헤더, URL, 쿠키 같은 내용을 보고 분배 대상을 결정합니다. URL 경로별 라우팅, 세션 고정, SSL 종료 등 더 세밀한 제어가 가능합니다.

주요 분배 알고리즘

알고리즘특징
라운드 로빈순서대로 균등 분배. 서버 성능이 동일할 때 적합
최소 연결현재 연결이 가장 적은 서버로 분배. 처리 시간 편차가 클 때 효과적
IP 해시클라이언트 IP로 항상 같은 서버에 연결. 세션 유지가 필요할 때

헬스 체크의 중요성

로드 밸런서는 주기적으로 백엔드 서버의 상태를 확인합니다. 헬스 체크에 실패한 서버는 자동으로 풀에서 제외되어 장애가 난 서버로 트래픽이 흐르지 않게 됩니다.

SSL 종료

L7 로드 밸런서에서 TLS를 종료하면 백엔드 서버들은 암호화/복호화 부담 없이 HTTP로 통신할 수 있습니다. 인증서 관리도 로드 밸런서 한 곳에서만 하면 됩니다.
