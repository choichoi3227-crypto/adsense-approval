리눅스 네트워크 상태 진단 명령어 모음

네트워크 문제를 만났을 때 어디서부터 확인해야 할지 막막하다면, 기본 명령어 몇 가지만 순서대로 실행해 보세요.

연결 상태 확인 — ss와 netstat

ss -tlnp는 현재 수신 대기(Listen) 중인 TCP 포트와 연결된 프로세스를 보여줍니다. netstat의 현대적 대체 명령어로, 더 빠르고 상세한 정보를 제공합니다.

경로 추적 — traceroute와 mtr

traceroute 목적지는 패킷이 목적지까지 거치는 라우터 경로를 추적합니다. mtr은 traceroute와 ping을 합친 도구로 각 홉의 패킷 손실률과 지연 시간을 실시간으로 보여줍니다.

DNS 조회 — dig와 nslookup

dig example.com A
dig @8.8.8.8 example.com MX

dig는 DNS 레코드를 상세히 조회할 수 있습니다. @서버로 조회할 DNS 서버를 지정할 수 있어, 설정한 DNS와 공개 DNS의 응답을 비교할 때 유용합니다.

패킷 캡처 — tcpdump

tcpdump -i eth0 port 80 -w capture.pcap

특정 인터페이스와 포트의 패킷을 캡처해 파일로 저장합니다. Wireshark로 열어 상세 분석할 수 있습니다. 문제 재현이 어려운 간헐적 네트워크 오류를 잡을 때 특히 유용합니다.

대역폭 측정 — iperf3

iperf3는 서버-클라이언트 쌍으로 실제 네트워크 처리량을 측정합니다. 서버 간 네트워크 성능을 검증하거나 병목 지점을 찾을 때 사용합니다.
