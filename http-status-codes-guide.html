HTTP 상태 코드 완전 정리: 1xx부터 5xx까지

상태 코드 숫자만 봐도 어디서 무슨 문제가 생겼는지 파악할 수 있다면 디버깅 속도가 크게 달라집니다.

2xx — 성공

코드의미
200 OK요청 성공
201 Created리소스 생성 성공 (POST)
204 No Content성공했지만 응답 본문 없음 (DELETE)

3xx — 리다이렉트

코드의미
301 Moved Permanently영구 이동. 검색엔진이 새 URL을 기억
302 Found임시 이동. 검색엔진은 원래 URL 유지
304 Not Modified캐시된 버전 사용 가능. 재다운로드 불필요

4xx — 클라이언트 오류

코드의미
400 Bad Request잘못된 요청 형식
401 Unauthorized인증 필요
403 Forbidden권한 없음 (인증과 무관)
404 Not Found리소스 없음
429 Too Many Requests요청 빈도 초과

5xx — 서버 오류

코드의미
500 Internal Server Error서버 내부 오류 (가장 흔한 서버 오류)
502 Bad Gateway프록시가 백엔드로부터 잘못된 응답 수신
503 Service Unavailable서버 과부하 또는 점검 중
504 Gateway Timeout프록시가 백엔드 응답 대기 시간 초과

운영 관점에서 주목할 코드

  • 502/504 급증: 백엔드 서버 문제 또는 타임아웃 설정 검토 필요
  • 429 증가: 비정상 트래픽 또는 rate limit 정책 조정 필요
  • 301 vs 302: SEO에 영향을 주므로 의도에 맞는 코드 사용 중요
